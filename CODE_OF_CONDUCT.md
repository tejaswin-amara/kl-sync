# Code of Conduct

This project is a community space. We're here to build something useful for KL University students, and we want it to be welcoming to everyone.

## Our values

- **Respect:** Treat everyone with kindness, regardless of background, identity, or experience level.
- **Honesty:** Be straightforward. Disagreements are fine — bad faith arguments are not.
- **Inclusion:** Avoid insider jargon without explanation. Help each other learn.
- **Safety:** Keep discussions focused on the project. Don't share personal information (real names, phone numbers, addresses) unless necessary.

## Expected behavior

- Ask good questions. "How do I set up kl-sync?" is welcome. "Why is this broken?" with no context is not.
- Give thoughtful feedback in code reviews and issue discussions.
- Acknowledge mistakes. "I didn't test this thoroughly" is better than silence.
- Help others. If you know the answer, share it.

## Unacceptable behavior

- Harassment, insults, or exclusionary comments based on identity
- Sharing others' private information (email, phone, address, credentials)
- Spam or repeated off-topic messages
- Attempts to exploit or sabotage the project
- Discrimination or gatekeeping ("You don't know X, so your opinion doesn't matter")

## If something goes wrong

- **In public:** flag it respectfully in the issue or PR. "This comment crosses a line" is better than escalating.
- **In private:** email `tejaswinamara@klh.edu.in` with details. Be specific about what happened and why it's a problem.
- **Expect action:** maintainers will investigate and take appropriate steps (warning, comment deletion, account suspension).

## Scope

This code of conduct applies to:
- GitHub issues, pull requests, and discussions
- Any official KL Sync communication channels (if we set up Discord, etc.)

It does *not* apply to behavior outside this project.

---

**Disagreements are healthy.** Rudeness is not. Let's build something great together.
